Aperçu

Présentation

Ce projet est né d'une volonté de simplifier le lancement de Rocket League au quotidien. Epic Games impose son propre client à chaque démarrage du jeu, même lorsque l'on ne souhaite pas interagir avec la boutique ou les autres services associés. L'objectif était de proposer une interface légère et directe, dédiée uniquement au lancement du jeu.

La particularité principale de ce launcher est sa gestion multi-comptes : chaque compte possède sa propre copie de l'exécutable Slipstream.exe, isolée dans un dossier dédié. Il est ainsi possible de basculer d'un compte à l'autre en quelques clics, sans jamais se reconnecter manuellement via Epic Games.

L'interface graphique est construite sur un Canvas Tkinter de 800×600 px avec image de fond personnalisable via un fichier background.png. La configuration des comptes est persistée localement dans un fichier config.json.

Fonctionnalités

Multi-comptes

Création de plusieurs comptes, chacun avec son propre Slipstream.exe isolé dans accounts/<nom>/.

Changement rapide

Dropdown custom pour basculer entre les comptes enregistrés en un seul clic.

Lancement direct

Lancement via subprocess.Popen avec CREATE_NO_WINDOW — aucune console visible sur Windows.

Suppression de compte

Suppression propre avec confirmation : nettoyage du dossier via shutil.rmtree et mise à jour de la config.

Fond personnalisable

Image de fond chargée et redimensionnée via Pillow depuis un fichier background.png remplaçable librement.

Config persistante

Comptes et dernier compte actif sauvegardés automatiquement dans config.json à chaque modification.

Stack technique

Le projet est entièrement écrit en Python sans dépendances lourdes. L'interface repose sur Tkinter avec un Canvas pour le rendu des boutons. Pillow gère le chargement de l'image de fond. La persistance est assurée par un fichier JSON et le lancement du jeu par subprocess avec le flag CREATE_NO_WINDOW.

Python Tkinter Pillow subprocess shutil JSON

Démarche

Identification de la solution de base

Le point de départ repose sur les travaux de jun-eau et LittleScripterBoy, qui ont développé Slipstream, un exécutable permettant de lancer Rocket League sans Epic Games.

Conception du système multi-comptes

Chaque compte est isolé dans son propre sous-dossier contenant une copie de Slipstream.exe. Lors de la création d'un compte, l'exécutable source est dupliqué via shutil.copy dans accounts/<nom>/.

Interface dessinée sur Canvas

L'interface est dessinée directement sur un Canvas de 800×600 px, permettant de superposer les éléments sur l'image de fond avec un contrôle total sur le positionnement.

Lancement du jeu et fermeture propre

Le lancement s'effectue via subprocess.Popen en passant le répertoire du compte comme cwd. La fenêtre du launcher se ferme ensuite automatiquement via root.destroy().

Utilisation

01

Placer Slipstream.exe et launcher.py dans le même dossier. Ajouter un fichier background.png pour personnaliser l'arrière-plan (optionnel).

02

Lancer launcher.py avec Python. Cliquer sur New Account et saisir un nom pour créer le premier compte.

03

Pour changer de compte, cliquer sur le nom du compte affiché pour ouvrir le dropdown et sélectionner un autre compte.

04

Cliquer sur Start pour lancer Rocket League. Le launcher se ferme automatiquement.

05

Pour supprimer un compte, le sélectionner puis cliquer sur 🗑️ et confirmer.
