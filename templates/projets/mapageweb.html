Présentation

L'objectif de ce projet est de réaliser une page web qui permet de centraliser l'ensemble de mes projets afin de servir de hub pour mes différents projets, ainsi que de servir de page de présentation de mes différentes réalisations.

Fonctionnalités

Utilisateurs

Gestion de plusieurs utilisateurs simultanément

Enregistrement Scores

Une page d'enregistrement de score dans plusieurs jeux de société

Dashboard Proxmox

Une page de Dashboard Proxmox afin de voir l'utilisation de mes ressources

Note

Une page gestion de notes par utilisateurs

Partage de fichier chiffrée

Une page afin de pouvoir partager des fichiers entre 2 utilisateurs de manière chiffrée de bout en bout

Une page de gestion de la base de données

Notification automatique pour certaines actions avec ntfy

Globalité

Dans ce projet, j'ai plusieurs composantes importantes. Tout d'abord, un serveur Flask qui sera utilisé pour le backend. J'ai également une base de données PostgreSQL qui est utilisée afin de stocker des informations sur les utilisateurs. Enfin, j'ai un service reverse proxy Nginx qui permet de faire le lien entre mon adresse IP publique et les différentes pages à ma disposition.

Utilisateurs

Cette page web propose un service de connexion sécurisée avec un identifiant et un mot de passe chiffré grâce à l'algorithme de hachage : bcrypt.

En effet, cet algorithme est utilisé car il possède un sel intégré, ce qui le protège contre les attaques par rainbow table. De plus, le hachage est volontairement lent, ce qui permet de résister contre les attaques par brute-force. Enfin, cet algorithme produira toujours une longueur fixe de caractères, ce qui ne permet pas de déterminer d'informations sur le mot de passe chiffré.

Il y a un souci principal : cet algorithme tronque les mots de passe à 72 caractères maximum, mais je considère qu'il est suffisant pour mon utilisation.

En plus de cette gestion multi-utilisateurs, j'ai également une gestion des droits par utilisateur. Ainsi, tous les utilisateurs n'ont pas accès à toutes les pages. Il m'est donc possible de garder des pages en privé ou de les partager uniquement à certains membres de ma famille (à condition que leur compte ne soit pas compromis).

Enregistrement scores

Étant donné que je suis un joueur de jeux de société, j'ai toujours aimé réaliser des tableaux Excel afin de noter les scores des différentes parties. Il m'a donc paru naturel de faire la même chose ici, sur mon propre site web.

Ainsi, j'ai créé une page recensant différents jeux de société. Pour chaque jeu, j'ai ajouté :

  • une page dédiée aux règles, afin de pouvoir les partager facilement avec de nouveaux joueurs ;
  • un espace de saisie des scores réalisés lors des différentes parties.

L'intégralité de ces scores est enregistrée dans ma base de données personnelle et me permet de réaliser des statistiques sur les parties (la personne qui gagne le plus, les points moyens par jeu, etc.).

Cela me permet d'atteindre 2 objectifs principaux :

  • garder un moyen simple et rapide de noter les scores pour les différents jeux de société ;
  • conserver mes données personnelles sans les confier aux GAFAM et autres plateformes tierces, même si les données partagées ici ne sont pas sensibles.

Dashboard Proxmox

Étant donné que j'héberge l'ensemble de mes services sur un serveur Proxmox, je souhaitais avoir un accès facile à un dashboard simplifié me permettant de visualiser ma consommation de RAM, CPU, etc. Cela me permet de déterminer si une amélioration des composants de mon serveur est nécessaire ou non. De plus, cela me permet d'effectuer des actions supplémentaires directement depuis mon serveur.

Pour réaliser cela, j'ai créé un nouvel utilisateur Proxmox disposant uniquement des accès aux données que je souhaite exposer. J'utilise ensuite un token d'authentification afin de récupérer les informations souhaitées directement depuis la machine, de manière sécurisée.

Note

Lors du début de ce projet, je n'avais pas d'objectifs particuliers. Cependant, au fur et à mesure de l'avancement, je me retrouvais avec de nombreuses idées sans endroit pour les noter. C'est ainsi qu'est née l'idée de créer une page de gestion de notes par utilisateur.

Je souhaitais que ces notes puissent être partagées avec d'autres utilisateurs. Il serait ainsi possible, par exemple, de créer une liste de courses visible par tous les membres d'une même famille et modifiable par chacun d'eux.

Pour réaliser cela, j'ai utilisé ma propre base de données avec :

  • une table dédiée aux notes ;
  • une table dédiée aux partages.

Suite aux retours de certains utilisateurs souhaitant un système de groupes de notes, j'ai implémenté cette fonctionnalité. Désormais, toutes les notes sont associées à un groupe, personnel par défaut, et peuvent être déplacées vers un autre groupe.

La limite principale est qu'une note ne peut appartenir qu'à un seul groupe à la fois, ce qui est un choix délibéré car la gestion multi-groupes ne semblait pas apporter de réelle valeur ajoutée.

Partage de fichiers

Je souhaitais également pouvoir réaliser du partage de fichiers chiffré de bout en bout, afin que deux utilisateurs puissent échanger des fichiers sensibles à distance (clés de chiffrement, documents confidentiels, etc.).

Pour cela, j'ai développé une page de partage de fichiers sécurisée. Le chiffrement repose sur le mécanisme suivant :

  • utilisation d'une paire de clés RSA propre à chaque utilisateur ;
  • génération collaborative d'une clé AES jetable, qui sert à chiffrer les fichiers.

Ce partage est véritablement chiffré de bout en bout, puisque même sur le serveur, les fichiers restent chiffrés et inaccessibles en clair.

Il est ensuite possible de partager ces fichiers avec un ou plusieurs utilisateurs, qui pourront les consulter et les télécharger en toute sécurité.

Base de données

Comme expliqué dans les sections précédentes, j'utilise très fréquemment des données persistantes. Afin de les stocker, j'utilise une base de données PostgreSQL, hébergée en local sur mon serveur dans une machine virtuelle dédiée.

L'utilisation de cette base de données est indépendante de la page web. Elle peut ainsi être utilisée pour l'ensemble de mes autres services et tâches, et non uniquement pour mon site web personnel.

Notifications automatiques

Afin de surveiller certains problèmes potentiels et d'éviter que des personnes malveillantes cherchent à nuire en saturant ma base de données ou en exploitant une faille de mon site web, j'ai mis en place plusieurs points de contrôle de sécurité qui s'activent automatiquement en cas d'incident.

Lors du déclenchement d'un de ces points de sécurité, une notification automatique est envoyée grâce à NTFY. Il s'agit d'un service de notifications sur lequel il est possible de s'abonner afin de recevoir une alerte pour chaque message publié. En utilisant un nom de canal privé, il m'est possible de recevoir ces notifications de manière confidentielle en m'abonnant à ce nom. Ce service fonctionne via des requêtes HTTP.
