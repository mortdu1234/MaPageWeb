Aperçu

Présentation

Ce projet est né d'une volonté d'automatiser des taches répétitives dans le jeu mobile Dragon City. En effet ce jeu propose de regarder 30 pubs toutes les heures afin de gagner des récompenses.

Pour créer ce script j'ai utiliser ADB et BlueStack afin de pouvoir interragir avec l'appareil android. Ce script vas rechercher sur l'ecran des patterns prédéfinis et quand il les trouve, vas simuler un clic dessus.

Fonctionnalités

Creation d'un nouveau template

Permet de créer des nouveau patterns de recherche.

Fermeture d'une pub

Effectue la fermeture d'une seule et unique pub

Lancement d'un cycle

Le programme vas effectuer un cycle de recherche de pub

Stack technique

Le projet est entièrement écrit en Python avec des dépendance importante avec ADB et un emulateur compatible avec ADB. Il propose une interface Terminale pour gerer ce programme et s'auto alimente de nouveau template avec les patterns assez fiable.

Python ADB Pillow subprocess shutil JSON

Démarche

Identification de la solution de base

Le point de départ repose sur les travaux de jun-eau et LittleScripterBoy, qui ont développé Slipstream, un exécutable permettant de lancer Rocket League sans Epic Games. L'étude de leur approche a permis de comprendre le mécanisme de lancement et ce qui était nécessaire pour le reproduire.

Conception du système multi-comptes

L'idée centrale est d'isoler chaque compte dans son propre sous-dossier contenant une copie de Slipstream.exe. Lors de la création d'un compte, l'exécutable source est dupliqué via shutil.copy dans accounts/<nom>/. Chaque instance conserve ainsi ses données de session sans conflit entre les comptes.

Interface dessinée sur Canvas

Plutôt que d'utiliser les widgets Tkinter classiques, l'interface est dessinée directement sur un Canvas de 800×600 px. Cela permet de superposer les éléments sur l'image de fond avec un contrôle total sur le positionnement. Chaque bouton est un rectangle Canvas avec un binding <Button-1> attaché à la fois au rectangle et au texte.

Gestion du dropdown et des clics extérieurs

Le menu de sélection de compte est un dropdown entièrement custom dessiné sur le Canvas. Les coordonnées des zones interactives sont stockées dans des variables globales et comparées à chaque clic via un binding add="+", ce qui permet de fermer le menu lors d'un clic extérieur sans interférer avec les autres bindings déjà enregistrés.

Lancement du jeu et fermeture propre

Le lancement s'effectue via subprocess.Popen en passant le répertoire du compte comme cwd, avec le flag CREATE_NO_WINDOW sur Windows pour éviter l'apparition d'une fenêtre console. La fenêtre du launcher se ferme ensuite automatiquement via root.destroy().

Utilisation

01

Placer Slipstream.exe et launcher.py dans le même dossier. Ajouter un fichier background.png pour personnaliser l'arrière-plan (optionnel).

02

Lancer launcher.py avec Python. Cliquer sur New Account et saisir un nom pour créer le premier compte.

03

Pour changer de compte, cliquer sur le nom du compte affiché pour ouvrir le dropdown et sélectionner un autre compte enregistré.

04

Cliquer sur Start pour lancer Rocket League avec le compte actif. Le launcher se ferme automatiquement.

05

Pour supprimer un compte, s'assurer qu'il est sélectionné puis cliquer sur 🗑️ et confirmer. Le dossier associé est supprimé proprement.
