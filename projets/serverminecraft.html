Présentation

Ce projet est une extension naturelle de mon serveur auto-hébergé. Disposant déjà d'une infrastructure Proxmox, j'ai voulu en profiter pour héberger un serveur Minecraft et jouer avec mes amis sans dépendre d'un hébergeur tiers.

J'ai créé une machine virtuelle dédiée sous Proxmox, installé Java et les fichiers du serveur Minecraft dessus. J'ai fait tourner deux types de serveurs selon les besoins : un serveur Vanilla (officiel Mojang, sans modification) et un serveur Forge pour jouer avec des mods. La principale difficulté a été de trouver un moyen simple et fiable de rendre ce serveur accessible depuis l'extérieur du réseau local.

Démarche

Création de la VM et installation du serveur

J'ai créé une machine virtuelle sous Proxmox, installé Java, puis déployé les fichiers du serveur Minecraft. Le serveur Vanilla a été mis en place en premier, puis un serveur Forge a été ajouté pour permettre de jouer avec des mods. Les deux instances tournaient sur la même VM, sur des ports différents.

Première tentative : Tailscale

Tailscale est un VPN mesh basé sur WireGuard qui crée un réseau privé entre plusieurs appareils via Internet, sans configuration de routeur. En théorie, mes amis auraient pu rejoindre le serveur comme s'ils étaient sur mon réseau local. En pratique, cela impliquait que chacun installe et configure Tailscale sur sa machine, ce qui s'est révélé trop contraignant pour une utilisation détendue entre amis.

Deuxième tentative : playit.gg

playit.gg est un service de tunnel qui expose un serveur local sur Internet via une adresse publique fournie par playit, sans avoir besoin de toucher au routeur ni d'ouvrir de port. Côté client, aucune installation n'est requise : une simple adresse IP suffit pour se connecter. C'était bien plus simple pour mes amis, mais la version gratuite imposait des limitations (nombre de tunnels, bande passante) qui ont fini par freiner l'usage au quotidien.

Bilan

Ce projet m'a permis d'explorer différentes approches pour exposer un service réseau sur Internet : VPN mesh, tunnel applicatif, et ouverture de port classique. Aucune solution ne s'est avérée entièrement satisfaisante dans ce contexte, et le serveur n'est plus actif aujourd'hui. L'expérience reste utile : ces mêmes problématiques d'exposition réseau se retrouvent dans tous mes projets d'auto-hébergement.

Stack technique

Le serveur tourne dans une machine virtuelle Linux sous Proxmox avec Java comme seule dépendance. Deux types de serveurs ont été déployés : le serveur Vanilla officiel de Mojang, et un serveur Forge pour la gestion des mods. L'accès externe a été tenté via Tailscale puis playit.gg.
